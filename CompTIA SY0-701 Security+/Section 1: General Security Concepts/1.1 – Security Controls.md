# 🔐 1.1 — Security Controls (Contrôles de sécurité)

---

## 📌 Définition

Un **Security Control** est une mesure (technique, humaine ou organisationnelle) mise en place pour :

- Prévenir une attaque
- Détecter une attaque
- Corriger une attaque

👉 C’est **la base de toute stratégie de cybersécurité**.

---

## 🧩 Les grandes catégories de contrôles

---

### 🛡️ 1. Preventive Controls (Contrôles préventifs)

👉 Empêchent une attaque avant qu’elle ne se produise.

**Exemples :**
- Firewall
- Authentification MFA
- ACL (Access Control Lists)
- Segmentation réseau

**💥 Exemple réel :**
Un firewall bloque une IP malveillante → l’attaque n’atteint jamais le serveur.

---

### 🔍 2. Detective Controls (Contrôles détectifs)

👉 Identifient une attaque en cours ou passée.

**Exemples :**
- Logs système
- SIEM
- IDS (Intrusion Detection System)

**💥 Exemple réel :**
Connexion suspecte à 3h du matin → alerte SIEM.

---

### 🔧 3. Corrective Controls (Contrôles correctifs)

👉 Réparent les dégâts après une attaque.

**Exemples :**
- Backup
- Patch management
- Antivirus

**💥 Exemple réel :**
Restauration d’un serveur après ransomware.

---

## 🧠 Autres classifications importantes (EXAM TRÈS IMPORTANT)

---

### 📊 4. Administrative Controls

👉 Basés sur les politiques et les humains.

**Exemples :**
- Security policies
- Formation employés
- Procédures internes

---

### ⚙️ 5. Technical Controls

👉 Implémentés via la technologie.

**Exemples :**
- Firewall
- IDS/IPS
- Antivirus

---

### 🏢 6. Physical Controls

👉 Protègent l’infrastructure physique.

**Exemples :**
- Caméras
- Badge RFID
- Serrures

---

## ⚠️ Piège classique à l’examen

👉 Un contrôle peut appartenir à **PLUSIEURS catégories**.

Exemple :
- Firewall = Preventive + Technical

---

## 🧠 Vision terrain (important pour toi)

En bug bounty / pentest :

- Preventive FAIL → vulnérabilité exploitable
- Detective FAIL → attaque invisible
- Corrective FAIL → dégâts permanents

👉 Une vraie sécurité = combinaison des 3
